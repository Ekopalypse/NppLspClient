
NppLspClient uses a JSON encoded configuration file.

To add an LSP server to this file, add a JSON object to the lspservers object.
Save and apply the changes.
This stops any LSP servers that may be running and the file is read in again.

The following values can be defined for a server instance:
	Name				Typ				Description
	pipe				string			Either io or tcp (NOTE: only io mode, reading/writing from/to stdin/stdout is implemented yet.)
	executable			string			The full path to language server executable
	args				string array	Parameters which should be passed to the lsp server executable
	auto_start_server	boolean			Experimentel, should be false for the time being.
	port 				int				TCP port used by the server, only needed in tcp mode.

For example:

		"python": {
			"pipe": "io",
			"executable": "D:\\Python\\Python38_64\\Scripts\\pyls.exe",
			"args": ["--check-parent-process", "--log-file", "D:\\log.txt", "-vvv"],
			"auto_start_server": false
		}

Note that the name of the instance, e.g. "python" must be the same as displayed in the Notepad++ language menu, but in lower case.

An example configuration with several LSP servers looks like this:

{
	"lspservers": {
		"python": {
			"pipe": "io",
			"executable": "D:\\Python\\Python38_64\\Scripts\\pyls.exe",
			"args": ["--check-parent-process", "--log-file", "D:\\log.txt", "-vvv"],
			"auto_start_server": false
		},
		"vlang": {
			"pipe": "io",
			"executable": "D:\\Repositories\\eko\\vls\\bin\\vls.exe",
			"args": ["--vroot=D:\\ProgramData\\Compiler\\v", "--debug"],
			"auto_start_server": false
		},
		"d": {
			"pipe": "io",
			"executable": "D:\\ProgramData\\Compiler\\d_lsp\\serve-d.exe",
			"args": [],
			"auto_start_server": false
		}
	}
}
